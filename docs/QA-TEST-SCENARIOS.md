## 5. 경기 생성 (Host)

> 폼 구조: 스텝 네비게이션 없이 **5개 섹션이 한 페이지에 나열**되는 싱글 폼.
> 제출 시 섹션별 유효성 검증 → 실패 시 해당 섹션으로 자동 스크롤.

### 5-1. 페이지 진입 & 공통 [P0]

| # | 사전조건 | 행동 | 기대결과 |
|---|---------|------|---------|
| 1 | 경기 생성 진입 | 페이지 로드 | 5개 섹션 한 페이지에 표시 (스텝 없음) |
| 2 | 첫 방문 | 팁 배너 확인 | "딱 한 번만 작성하세요!" 온보딩 팁 표시 |
| 3 | 팁 배너 | X 클릭 | 배너 닫힘, localStorage에 저장 (다음 방문 시 안 뜸) |
| 4 | 헤더 | "최근 경기 불러오기" 버튼 클릭 | 최근 경기 다이얼로그 → 선택 시 폼 프리필 |
| 5 | 최근 경기 프리필 후 | 토스트 확인 | "지난 경기 정보를 불러왔습니다. 경기 날짜를 선택해주세요." |

### 5-2. 섹션 1: 기본 정보 + 시설 [P0]

| # | 사전조건 | 행동 | 기대결과 |
|---|---------|------|---------|
| 1 | 기본 정보 | 제목 입력 (1~100자) | 유효 |
| 2 | 기본 정보 | 날짜 선택 (14일 캘린더) | 선택 반영 |
| 3 | 기본 정보 | 시작 시간 선택 | 선택 반영 |
| 4 | 기본 정보 | 소요 시간 선택 (1h~4h) | 종료 시간 자동 계산 |
| 5 | 기본 정보 | 장소 검색 (카카오 맵) | 검색 결과 드롭다운, 선택 시 주소 자동입력 |
| 6 | 기존 체육관 선택 | 장소 선택 후 | 시설 정보(주차/냉난방/샤워/공 등) 자동 프리필 |
| 7 | 기본 정보 | 참가비 유형: 금액/음료 선택 | 금액 시 원 단위, 음료 시 병 수 |
| 8 | 시설 정보 | 주차: 무료/유료/불가 선택 | 유료 시 상세(주차 위치) 입력 |
| 9 | 시설 정보 | 냉난방/샤워/음수대/공/음료 토글 | 각 ON/OFF |
| 10 | 시설 정보 | 코트 크기: 정규/숏/좁음 선택 | 선택 반영 |
| 11 | 제출 시 | 날짜 미선택 | 에러 토스트 + 기본 정보 섹션으로 스크롤 |
| 12 | 제출 시 | 장소 미선택 (카카오 API 미사용) | 에러 토스트 + 기본 정보 섹션으로 스크롤 |

## 테스트 진행 가이드

### 권장 순서
1. **빌드 확인**: `npm run build` 통과
2. **P0 시나리오**: 인증 → 경기 신청 → 경기 관리 → 결제 흐름 → 계정 탈퇴
3. **P1 시나리오**: 필터, 팀, 프로필, 알림, 알림 설정, 취소 흐름
4. **P2~P3 시나리오**: 시설, 공유, UI/UX 세부

### 테스트 환경
- 모바일 뷰포트 (430px 이하)
- Chrome DevTools 모바일 시뮬레이션 권장
- 계정 2개 준비 (게스트용 + 호스트용)
