## 5. Detailed User Flow (Approval Process)

This platform follows a "Trust-Based Approval" model.

### Phase 1: Application (Guest)
1.  **Discover**: Guest finds a match on the Home Feed (`/`). Cards show time, location, and vacancy badges.
2.  **Apply**: Guest clicks into detail page (`/guest/[id]`). Checks position requirements.
3.  **Action**: Clicks "Apply".
4.  **State Change**: Status becomes `pending`. (Host sees new applicant).

### Phase 2: Approval (Host)
1.  **Notification**: Host checks Dashboard (`/host/dashboard`).
2.  **Review**: Host reviews Guest's Manner Score and Position.
3.  **Action**: Host clicks "Approve".
4.  **State Change**: Status becomes `approved_pending_payment`.
5.  **Timer**: A 1-hour payment timer starts (Mock logic).

### Phase 3: Payment (Guest)
1.  **Verification**: Guest sees "Payment Required" status.
2.  **Transfer**: Guest sends money to Host's account (info in drawer).
3.  **Action**: Guest clicks "Payment Complete".
4.  **Warning**: System shows "False Report Warning" Modal. Guest confirms.
5.  **State Change**: Status becomes `verification_pending`.

### Phase 4: Confirmation (Host)
1.  **Notification**: Host Dashboard shows applicant with "Check Payment" badge (Orange).
2.  **Check**: Host verifies real bank app for transfer.
3.  **Action**:
    -   If Received: Click "Confirm". Status -> `confirmed` (Final).
    -   If Not Received: Click "Report". Status -> `rejected` + Manner Score Penalty.
